# مرحلة 1 - Foundation and Cloud Integration

## هدف المرحلة

تجهيز المشروع ليتحول من:

`keywords -> domains -> scoring -> review`

إلى نواة صالحة للتوسع لاحقًا نحو:

`signals -> processing -> intelligence -> domain opportunities -> scoring -> shortlist`

بدون كسر التطبيق الحالي أو إعادة كتابة نظام التقييم الموجود.

## المطلوب منك لتفعيل الأجزاء الخارجية

### 1. لتفعيل Supabase

أحتاج منك:

- `SUPABASE_URL`
- `SUPABASE_KEY`

الخطوات:

1. إنشاء مشروع على Supabase
2. فتح SQL Editor
3. تشغيل الملف `supabase/schema.sql`
4. وضع القيم في `.env` أو `Streamlit secrets`
5. ضبط `USE_SUPABASE=true`

### 2. لتفعيل LLM providers

أحتاج منك حسب المزود الذي تريد استخدامه:

- `XAI_API_KEY`
- أو `GEMINI_API_KEY`
- أو `OPENROUTER_API_KEY`

واختياريًا:

- `XAI_MODEL`
- `GEMINI_MODEL`
- `OPENROUTER_MODEL`

الخطوات:

1. إضافة المفاتيح في `.env` أو `Streamlit secrets`
2. تشغيل التطبيق
3. اختبار الاتصال من الـ sidebar

### 3. لتفعيل Namecheap

أحتاج منك:

- `NAMECHEAP_API_USER`
- `NAMECHEAP_API_KEY`
- `NAMECHEAP_USERNAME`

## التحقق الذي تم

تم التحقق من:

- نجاح `compileall` باستخدام `.venv\Scripts\python.exe`
- سلامة البناء للطبقات الجديدة

ملاحظة:

اختبار runtime الكامل داخل البيئة الحالية توقف عند نقص بعض dependencies مثل `pandas` داخل `.venv` الحالية، وليس بسبب Syntax Error في الكود.
