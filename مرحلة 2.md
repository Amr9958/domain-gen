# مرحلة 2 - Collection + Processing

## الوضع الحالي للمرحلة

Phase 2 لم تعد مجرد دفعة أولى. المرحلة الآن وصلت إلى شكل عملي أوضح يشمل:

- جمع raw signals من المصادر الخارجية
- فصل ingestion عن processing في jobs مستقلة
- تنظيف وتطبيع وإزالة تكرار
- clustering أقوى من النسخة الأولى مع theme consolidation مبدئية بين runs
- classification heuristics أولية
- theme extraction مبدئي
- keyword intelligence مبدئية
- domain opportunity generation أقوى من النسخة الأولى
- shortlist / watchlist / rejected workflows
- retry/backoff أساسي للمصادر الخارجية
- structured source tagging داخل themes
- source-aware exact-match / trademark safety
- حفظ أوضح بين raw وprocessed

## خطوات التشغيل

### تشغيل ingestion فقط

```bash
python3 jobs/ingest_signals.py
```

### تشغيل processing فقط

```bash
python3 jobs/process_signals.py
```

### تشغيل الـ pipeline كاملة كما كان سابقًا

```bash
python3 jobs/collect_signals.py
```

هذا الأمر أصبح الآن يشغّل:

- `ingest`
- `process`
- `generate_domain_ideas`

### تشغيل domain opportunity generation

بعد إنهاء `process_signals.py`:

```bash
python3 jobs/generate_domain_ideas.py
```

### في حالة عدم تفعيل Supabase

سيتم الحفظ محليًا داخل `DATA_DIR/signals`.

### في حالة تفعيل Supabase

تأكد أولًا من تشغيل:

- `supabase/schema.sql`

وإذا كنت قد شغّلت schema القديمة سابقًا، شغّل النسخة الحالية لأنها تحتوي أيضًا على `ALTER TABLE` لإضافة الأعمدة الجديدة.

تم التحقق من:

- ترابط collectors/processors/jobs على مستوى البنية

- compile وimport للملفات الجديدة والمعدلة
- smoke test على توليد keyword insights وdomain opportunities
- ربط النتائج داخل `app.py` بدون كسر التبويبات الحالية
- smoke test على theme consolidation مع historical theme
- smoke test على review buckets مثل `shortlist`
- smoke test على structured source tagging داخل themes
- smoke test على خفض score/review bucket عند overlap مع source-specific entity

## ما الذي ما زال ناقصًا بعد هذا التحديث

ما زال هناك جزء لاحق قبل اعتبار intelligence layer مكتملة بالكامل:

- keyword intelligence أذكى وربط أقوى بين themes المتقاربة جدًا
- domain opportunity generation أدق أكثر في trademark-awareness عبر مصادر أوسع وقوائم أفضل
- rate-limit safety أوسع مع config أكثر وtelemetry أوضح
- dashboard أوسع من تبويب واحد إلى صفحات أو workflows أعمق
- selective LLM refinement بعد الفلترة النهائية

## الخطوة التالية المنطقية

الخطوة التالية بعد هذا التحديث أصبحت أقرب إلى shortlist intelligence الفعلية:

1. توسيع trademark-awareness بقوائم أوسع وإشارات richer من المصادر
2. إضافة source-driven explanation أعمق داخل review UI
3. إضافة shortlist refinement اختياري بالـ LLM بعد الفلترة
