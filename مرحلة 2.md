# مرحلة 2 - الدفعة الأولى

## ما تم في هذه الدفعة

تم بدء Phase 2 فعليًا بأول pipeline كاملة لجمع الإشارات ومعالجتها بشكل مبدئي.

### Collectors المضافة

- `collectors/hackernews.py`
- `collectors/github.py`
- `collectors/gnews.py`
- `collectors/base.py`

المصادر المدعومة الآن:

- Hacker News
- GitHub Search API
- GNews

### Processors المضافة

- `processors/cleaning.py`
- `processors/dedup.py`
- `processors/clustering.py`

الذي يتم الآن:

- تنظيف النص
- تطبيع العناوين والمحتوى
- إزالة التكرار
- clustering خفيف يعتمد على أكثر الكلمات تكرارًا

### Persistence للـ signal pipeline

تمت إضافة:

- `repositories/signals.py`

وسلوك الحفظ الآن:

- إذا كان Supabase مفعلًا: يتم الحفظ هناك
- إذا لم يكن مفعلًا: يتم الحفظ محليًا كـ JSONL داخل `data/signals` فعليًا تحت `DATA_DIR/signals`

### Job Entrypoint

تمت إضافة:

- `jobs/collect_signals.py`

وهذا job يقوم بـ:

1. جمع البيانات من collectors
2. تنظيفها
3. إزالة التكرار
4. عمل clustering مبدئي
5. حفظ raw items
6. حفظ processed items
7. حفظ run metadata

## المطلوب منك لتفعيل أفضل

### 1. GitHub API

اختياري لكنه مفيد جدًا لتقليل limits:

- `GITHUB_TOKEN`

### 2. GNews

لتشغيل collector الخاص به:

- `GNEWS_API_KEY`

اختياريًا:

- `GNEWS_LANG`
- `GNEWS_COUNTRY`

### 3. Supabase

إذا أردت أن تُخزّن الـ signals سحابيًا بدل local JSONL:

- `SUPABASE_URL`
- `SUPABASE_KEY`
- `USE_SUPABASE=true`

## ما تم التحقق منه

تم التحقق من:

- compile للملفات الجديدة
- ترابط collectors/processors/jobs على مستوى البنية

## ما الذي ما زال ناقصًا في Phase 2

هذه ليست نهاية Phase 2 بعد. ما زال مطلوبًا في الدفعات التالية:

- تحسين clustering
- source tagging أعمق
- classification heuristics
- persistence أدق بين raw وprocessed
- تحسين جداول Supabase لتفريق raw عن normalized
- rate-limit safety أوسع
- ربما job منفصلة للـ process بدل الجمع والمعالجة في خطوة واحدة

## الخطوة التالية المقترحة

الدفعة التالية في Phase 2 ستكون:

1. فصل raw ingestion عن processing في job مستقلة
2. إضافة classification أولية:
   - Investable
   - Watchlist
   - Low-value
   - Ignore
3. تحسين clustering وربط themes مبدئيًا
